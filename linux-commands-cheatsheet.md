# 📘 Шпаргалка по командам Linux для администраторов и разработчиков

Добро пожаловать в структурированный справочник по наиболее полезным командам Linux.  
Материал разбит на тематические разделы, каждая команда снабжена кратким русским пояснением.  
Подходит для ежедневной работы, решения проблем и подготовки к сертификациям (LPIC, RHCSA и др.).

---

## 🧠 Система: состояние и производительность (System Health & Performance)

Общие команды для оценки загрузки системы, информации о ядре, дистрибутиве, процессах и ресурсах.

| Команда | Описание |
|---------|----------|
| `uptime` | Показывает текущее время, как долго система работает, количество пользователей и среднюю загрузку за 1, 5 и 15 минут. |
| `hostnamectl` | Отображает или изменяет имя хоста и информацию о системе (версия ОС, ядро, архитектура). |
| `uname -a` | Выводит полную информацию о ядре Linux (имя хоста, версия ядра, дата сборки, архитектура). |
| `lsb_release -a` | Показывает данные о дистрибутиве (название, версию, кодовое имя) — если установлен пакет `lsb-release`. |
| `whoami` | Выводит имя текущего пользователя. |
| `id` | Показывает UID, GID и группы текущего пользователя. |
| `date` | Текущая дата и время в системе. |
| `timedatectl` | Управление системным временем, часовым поясом и синхронизацией NTP. |
| `free -h` | Использование оперативной памяти и swap в человекочитаемом виде (`-h`). |
| `vmstat 1` | Выводит статистику по процессам, памяти, подкачке, вводу-выводу и CPU (обновление раз в 1 секунду). |
| `top` | Интерактивный монитор процессов — нагрузка на CPU, память, запущенные задачи. |
| `htop` | Улучшенная версия `top` с цветами, прокруткой и управлением мышью (устанавливается отдельно). |
| `ps aux` | Список всех запущенных процессов с подробными атрибутами (пользователь, CPU, память, команда). |
| `pgrep nginx` | Находит PID процесса по имени (например, `nginx`). |
| `pidof sshd` | Возвращает PID процесса с точным именем (`sshd`). |
| `kill PID` | Отправляет процессу сигнал SIGTERM (корректное завершение). |
| `kill -9 PID` | Принудительное убийство процесса (сигнал SIGKILL) — использовать с осторожностью! |
| `pkill firefox` | Убивает все процессы, имя которых соответствует шаблону `firefox`. |
| `lscpu` | Детальная информация о процессоре: модель, ядра, потоки, частота, кэш. |
| `systemd-analyze blame` | Показывает время загрузки каждого системного сервиса — помогает найти медленные службы. |

## 🌐 Сеть и диагностика DNS (Network & DNS Fixes)

Команды для проверки сетевых интерфейсов, маршрутизации, соединения с интернетом и работы DNS.

| Команда | Описание |
|---------|----------|
| `ip a` | Показывает все сетевые интерфейсы и их IP-адреса (аналог `ifconfig`). |
| `ip route` | Таблица маршрутизации (как `route -n`). |
| `ping 8.8.8.8` | Проверка доступности узла (Google DNS) — базовый тест сети. |
| `ping google.com` | Проверка работы DNS и соединения с доменом. |
| `traceroute google.com` | Трассировка маршрута до узла (показывает все промежуточные хопы). |
| `ss -tulnp` | Список слушающих (listening) портов TCP и UDP с указанием программ. |
| `curl -I URL` | Получить только HTTP-заголовки ответа от сервера. |
| `wget --spider URL` | Проверить доступность URL без скачивания содержимого. |
| `nmcli dev status` | Статус сетевых устройств, управляемых NetworkManager. |
| `cat /etc/resolv.conf` | Посмотреть текущие DNS-серверы, используемые системой. |
| `resolvectl status` | Состояние DNS-резолвера systemd-resolved. |
| `resolvectl query google.com` | Выполнить DNS-запрос через systemd-resolved. |
| `dig google.com` | Расширенный DNS-запрос (тип записи, время ответа, полные данные). |
| `nslookup google.com` | Классический инструмент для простого DNS-запроса. |
| `ip neigh` | ARP-таблица — соответствие IP и MAC-адресов в локальной сети. |
| `sudo ip link set eth0 up` | Включить сетевой интерфейс `eth0`. |
| `sudo dhclient -v eth0` | Запросить новый IP-адрес по DHCP для интерфейса `eth0`. |
| `curl ifconfig.me` | Узнать внешний (публичный) IP-адрес через сторонний сервис. |
| `sudo systemctl restart NetworkManager` | Перезапустить службу управления сетями (осторожно — разорвёт соединения). |
| `sudo systemctl restart systemd-resolved` | Перезапустить локальный DNS-резолвер. |

## ⚙️ Сервисы, автозагрузка и инициализация (Services, Boot & Startup)

Работа с systemd: управление службами, просмотр логов, настройка режимов загрузки и заданий cron.

| Команда | Описание |
|---------|----------|
| `systemctl status nginx` | Показать состояние сервиса `nginx` (запущен/остановлен, PID, последние логи). |
| `sudo systemctl start nginx` | Запустить сервис. |
| `sudo systemctl stop nginx` | Остановить сервис. |
| `sudo systemctl restart nginx` | Перезапустить сервис (если не помогает `reload`). |
| `sudo systemctl reload nginx` | Перечитать конфигурацию без остановки (для сервисов, поддерживающих перезагрузку). |
| `sudo systemctl enable nginx` | Добавить сервис в автозагрузку (запуск при старте системы). |
| `sudo systemctl disable nginx` | Убрать из автозагрузки. |
| `systemctl list-units --failed` | Список сервисов, которые не удалось запустить (аварийное состояние). |
| `sudo systemctl reset-failed` | Сбросить флаг «failed» для всех сервисов. |
| `sudo systemctl daemon-reload` | Перечитать все unit-файлы systemd (после ручного изменения). |
| `journalctl -u nginx` | Показать логи только для сервиса `nginx`. |
| `journalctl -xe` | Последние системные сообщения с пояснениями (удобно для поиска ошибок). |
| `journalctl -p err -b` | Ошибки (`err`) за текущую сессию загрузки (`-b`). |
| `last reboot` | История перезагрузок системы (время, длительность работы). |
| `systemctl get-default` | Текущий target по умолчанию (графический `graphical.target` или текстовый `multi-user.target`). |
| `sudo systemctl set-default multi-user.target` | Переключить загрузку в текстовый режим (без графики). |
| `sudo systemctl set-default graphical.target` | Включить графический интерфейс при загрузке. |
| `systemctl list-timers` | Список всех таймеров systemd (аналог cron). |
| `crontab -l` | Просмотр заданий cron для текущего пользователя. |
| `sudo reboot` | Перезагрузить систему (требует прав, осторожно!). |

## 💾 Диски, файловые системы и хранилище (Disk, Filesystem & Storage)

Команды для оценки занятого места, работы с разделами, монтированием и ремонтом ФС.

| Команда | Описание |
|---------|----------|
| `df -h` | Использование дисковых разделов в удобном формате (размер, занято, доступно). |
| `du -sh *` | Размеры папок и файлов в текущей директории. |
| `du -ah . | sort -h` | Найти самые большие файлы рекурсивно (сортировка по размеру). |
| `lsblk` | Список блочных устройств (диски и их разделы) с размерами и точками монтирования. |
| `blkid` | UUID и типы файловых систем для каждого раздела. |
| `findmnt` | Древовидный вывод смонтированных файловых систем. |
| `mount` | Все текущие точки монтирования и параметры. |
| `sudo mount -a` | Смонтировать все файловые системы из `/etc/fstab` (проверка синтаксиса). |
| `cat /etc/fstab` | Файл, определяющий, какие диски монтировать при загрузке. |
| `sudo umount /mnt/disk` | Размонтировать раздел (осторожно — не размонтирует, если используется). |
| `journalctl --disk-usage` | Размер логов journald на диске. |
| `sudo journalctl --vacuum-time=7d` | Очистить логи старше 7 дней. |
| `sudo apt clean` | Удалить кэш скачанных пакетов (освобождает место). |
| `find /var/log -type f -size +100M` | Найти файлы логов больше 100 МБ. |
| `stat file.txt` | Подробная метаинформация о файле (размер, права, временные метки). |
| `file unknown.bin` | Определить тип файла (бинарный, текст, изображение и т.д.). |
| `ls -lah` | Подробный список файлов (включая скрытые) с размерами в человекочитаемом виде. |
| `sudo fsck -f /dev/sdXN` | Проверка и попытка исправления файловой системы (только для размонтированных разделов, осторожно!). |
| `sudo e2fsck -f /dev/sdXN` | Специальная проверка для ext2/ext3/ext4 (принудительная, даже если чистая). |
| `sudo tune2fs -l /dev/sdXN` | Детальная информация о ext-файловой системе (версия, метка, количество монтирований). |

## 📦 Пакеты, пользователи и права (Packages, Users & Permissions)

Управление программным обеспечением (apt), настройка прав доступа, добавление пользователей в группы.

| Команда | Описание |
|---------|----------|
| `sudo apt update` | Обновить список доступных пакетов из репозиториев. |
| `sudo apt upgrade` | Установить все обновления для уже установленных пакетов. |
| `sudo apt --fix-broken install` | Исправить проблемы с зависимостями (неудачные установки). |
| `sudo dpkg --configure -a` | Донастроить прерванные установки deb-пакетов. |
| `apt list --upgradable` | Показать пакеты, для которых есть обновления. |
| `apt policy package` | Узнать версию установленного пакета и доступные версии. |
| `sudo apt install package` | Установить пакет. |
| `sudo apt remove package` | Удалить пакет (конфигурационные файлы остаются). |
| `sudo apt autoremove` | Удалить пакеты, установленные как зависимости и больше не нужные. |
| `which command` | Показать полный путь к исполняемому файлу команды. |
| `ls -l` | Посмотреть права доступа к файлам (владелец, группа, чтение/запись/выполнение). |
| `namei -l /path/file` | Показать права на каждый компонент пути (полезно для поиска проблем доступа). |
| `chmod 644 file` | Установить стандартные права на файл: rw-r--r-- (владелец читает/пишет, остальные только читают). |
| `chmod 755 script.sh` | Права для скрипта: rwxr-xr-x (владелец может выполнять, остальные — только читать/выполнять). |
| `chmod +x script.sh` | Добавить флаг выполнения для всех пользователей. |
| `chown user:user file` | Сменить владельца и группу файла. |
| `groups username` | Показать группы, в которые входит пользователь. |
| `sudo usermod -aG group user` | Добавить пользователя в группу (опция `-a` важна — добавляет, не заменяя). |
| `sudo visudo` | Безопасное редактирование файла `/etc/sudoers` (с проверкой синтаксиса). |
| `passwd` | Сменить пароль текущего пользователя. |

## 📜 Логи и поиск ошибок (Logs & Error Hunting)

Поиск проблем через journald, syslog, dmesg и утилиты фильтрации.

| Команда | Описание |
|---------|----------|
| `sudo journalctl -b` | Логи текущей загрузки (с момента включения). |
| `journalctl -b -1` | Логи предыдущей загрузки (если сохранены). |
| `journalctl -f` | Следить за логами в реальном времени (tail -f для journald). |
| `journalctl --since "1 hour ago"` | Логи за последний час. |
| `journalctl -k` | Только сообщения ядра (kernel). |
| `dmesg -T` | Кольцевой буфер ядра с человекочитаемыми временными метками. |
| `dmesg -T | grep -i error` | Найти ошибки ядра. |
| `tail -n 100 /var/log/syslog` | Последние 100 строк системного журнала (для Debian/Ubuntu). |
| `tail -f /var/log/syslog` | Мониторинг syslog в реальном времени. |
| `grep -Ri "failed" /var/log` | Рекурсивный поиск слова "failed" по всем файлам в `/var/log`. |
| `grep -Ri "error" /var/log` | Поиск ошибок по всем логам. |
| `zgrep -i error /var/log/syslog*.gz` | Поиск в сжатых (ротированных) логах. |
| `less /var/log/syslog` | Постраничный просмотр лога с возможностью поиска (`/`). |
| `ls -lh /var/log` | Размеры файлов логов. |
| `sudo du -sh /var/log/*` | Оценка объёма каждой папки/файла в `/var/log`. |
| `sudo logrotate -d /etc/logrotate.conf` | Тестирование конфигурации ротации логов (без реального выполнения). |
| `sudo logrotate -f /etc/logrotate.conf` | Принудительно выполнить ротацию (осторожно — может сжать нужные логи). |
| `logger "HexSec log test"` | Отправить тестовое сообщение в системный журнал. |
| `sudo less /var/log/kern.log` | Просмотр лога ядра (обычно содержит те же сообщения, что и dmesg). |
| `sudo less /var/log/auth.log` | Лог аутентификации: входы в систему, sudo, ssh, pam. |

## 🖥️ Оборудование, драйверы и устройства (Hardware, Drivers & Devices)

Информация о PCI, USB, модулях ядра, звуке, питании, температуре и событиях устройств.

| Команда | Описание |
|---------|----------|
| `lspci` | Список всех устройств на шине PCI (видеокарты, сетевые карты, контроллеры). |
| `lsusb` | Список USB-устройств. |
| `sudo lshw -short` | Краткая сводка по всему оборудованию (требует прав). |
| `sudo lshw -C network` | Информация только о сетевых устройствах. |
| `lsmod` | Загруженные в ядро модули (драйверы). |
| `modinfo module` | Информация о модуле: версия, зависимости, параметры. |
| `sudo modprobe module` | Загрузить модуль в ядро. |
| `sudo modprobe -r module` | Выгрузить модуль (осторожно — может сделать устройство недоступным). |
| `rfkill list` | Состояние блокировки беспроводных интерфейсов (Wi-Fi, Bluetooth). |
| `sudo rfkill unblock all` | Разблокировать все радио-устройства. |
| `xrandr` | Вывести текущую конфигурацию экранов (разрешения, частоты, подключённые мониторы). |
| `xrandr --auto` | Автоматически настроить вывод на подключённые мониторы. |
| `aplay -l` | Список звуковых карт и их устройств. |
| `pactl info` | Информация о звуковом сервере PulseAudio. |
| `pactl list short sinks` | Список аудиовыходов (sinks) в PulseAudio. |
| `sensors` | Температуры процессора, материнской платы, жёстких дисков (требуется `lm-sensors`). |
| `upower -e` | Перечислить устройства питания (батареи, ИБП). |
| `upower -i DEVICE` | Детальная информация о батарее (ёмкость, состояние, заряд). |
| `udevadm monitor` | Следить за событиями подключения/отключения устройств (режим отладки). |
| `sudo udevadm trigger` | Принудительно пересканировать устройства и повторно применить правила udev. |

## 🔐 SSH, брандмауэр и удалённый доступ (SSH, Firewall & Remote Access)

Диагностика SSH-соединений, управление ключами, настройка UFW, firewalld, iptables и Fail2ban.

| Команда | Описание |
|---------|----------|
| `ssh -v user@host` | Соединение с SSH-сервером в режиме подробного вывода (отладка). |
| `ssh -p 2222 user@host` | Подключение к нестандартному порту (2222 вместо 22). |
| `ssh-keygen -R host` | Удалить из `known_hosts` ключ хоста (если изменился и ssh ругается). |
| `ssh-copy-id user@host` | Скопировать публичный ключ на сервер для входа без пароля. |
| `sudo sshd -t` | Проверить синтаксис конфигурационного файла SSH-сервера (`/etc/ssh/sshd_config`). |
| `sudo systemctl status ssh` | Состояние службы SSH-сервера. |
| `sudo systemctl restart ssh` | Перезапустить SSH-сервер (осторожно — активные соединения разорвутся). |
| `ss -tlnp | grep :22` | Проверить, слушает ли процесс порт 22 (и какой именно). |
| `sudo ufw status verbose` | Состояние брандмауэра UFW с деталями. |
| `sudo ufw allow 22/tcp` | Разрешить входящие соединения на TCP-порт 22 (SSH). |
| `sudo ufw deny 22/tcp` | Заблокировать порт 22 (осторожно — можно потерять доступ по SSH). |
| `sudo ufw reload` | Перезагрузить правила UFW. |
| `sudo firewall-cmd --state` | Статус firewalld (активен/неактивен). |
| `sudo firewall-cmd --list-all` | Все правила firewalld для зоны по умолчанию. |
| `sudo firewall-cmd --reload` | Перезагрузить правила firewalld без потери постоянных правил. |
| `sudo nft list ruleset` | Показать все правила nftables (современная замена iptables). |
| `sudo iptables -L -n -v` | Список правил iptables с числовыми адресами и счётчиками. |
| `sudo fail2ban-client status` | Общее состояние Fail2ban (защита от брутфорса). |
| `sudo fail2ban-client status sshd` | Статус jail для SSH: сколько заблокированных IP. |
| `nc -vz host 22` | Проверить, открыт ли порт 22 на удалённом хосте (TCP-соединение). |

## 💽 Резервное копирование, синхронизация и восстановление (Backup, Sync & Recovery)

Основы rsync, архивация tar/zip, проверка целостности, поиск изменённых файлов и атрибуты.

| Команда | Описание |
|---------|----------|
| `rsync -avh source/ dest/` | Синхронизация папок с архивным режимом (`-a`), подробностями (`-v`) и человекочитаемыми размерами (`-h`). |
| `rsync -avh --dry-run source/ dest/` | Просмотр, что будет скопировано (без реальных изменений). |
| `rsync -azh --progress source/ dest/` | Синхронизация со сжатием (`-z`) и индикатором прогресса. |
| `cp -a source dest` | Рекурсивное копирование с сохранением прав, владельцев, временных меток. |
| `cp file file.bak` | Быстрое создание локальной резервной копии файла. |
| `tar -czf backup.tar.gz folder/` | Создать сжатый gzip архив папки. |
| `tar -xzf backup.tar.gz` | Распаковать архив tar.gz в текущую папку. |
| `tar -tf backup.tar.gz` | Посмотреть содержимое архива без извлечения. |
| `zip -r backup.zip folder/` | Создать zip-архив папки. |
| `unzip backup.zip` | Распаковать zip. |
| `sha256sum file` | Вычислить SHA256-хеш файла (проверка целостности). |
| `md5sum file` | MD5-хеш (менее безопасен, но быстрее). |
| `diff -ru old/ new/` | Рекурсивное сравнение двух папок с выводом унифицированных отличий. |

## 📌 Заметки и рекомендации

- **Осторожность**: команды с пометкой `(caution)` могут нарушить работу системы или привести к потере данных (особенно `rm`, `dd`, `kill -9`, изменение `iptables/ufw` при удалённом доступе).
- **Права sudo**: большинство команд, изменяющих состояние системы, требуют привилегий root.
- **Интерактивные команды** (`top`, `htop`, `less`, `vim`) управляются клавишами (`q` для выхода, `/` для поиска и т.д.).
- **Пути к логам** могут отличаться в разных дистрибутивах (вместо `/var/log/syslog` на RHEL используется `/var/log/messages`).
